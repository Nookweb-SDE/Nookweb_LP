# Skill: Workflow & Memória de Sessão

## Antes de qualquer alteração
1. Ler o arquivo completo (ou trecho relevante) — NUNCA editar às cegas
2. Verificar se há outros arquivos que dependem do que será alterado
3. Se a mudança for grande ou impactar múltiplos arquivos → usar /plano primeiro

## Durante a alteração
- Alterar somente o necessário — não "melhorar" coisas fora do escopo
- Preservar estilo do projeto (indentação, nomenclatura, organização)
- Comentar apenas o necessário, não encher o código de comentários óbvios

## Após qualquer alteração
- Reler o trecho modificado
- Verificar imports, exports, referências CSS
- Resumir em 2–3 linhas o que foi feito e por quê

## Memória de contexto
- Manter lista mental do que já foi decidido nesta sessão
- Se o usuário já disse "não quero X", não sugerir X novamente
- Se algo foi feito e revertido, lembrar do motivo antes de sugerir de novo

## Preview visual
- Após mudanças visuais: tirar screenshot com /preview para mostrar resultado
- Comparar com estado anterior quando relevante
- Apontar se algo visualmente quebrou

## Comunicação
- Respostas diretas, sem enrolação
- Código primeiro, explicação depois
- Se não souber: "vou verificar" antes de responder
- Sempre em português
